## ⚠️ Restrição crítica da Onda 2 — ffmpeg.wasm SINGLE-THREAD

A versão **multithread** do ffmpeg.wasm exige *cross-origin isolation* (headers
`COOP: same-origin` + `COEP: require-corp`). Esses headers **quebram recursos cross-origin —
incluindo Google AdSense e analytics**, centrais para a monetização. Portanto:
- Usar **somente o core single-thread** (`@ffmpeg/core`, **não** `@ffmpeg/core-mt`).
- **Não** adicionar headers COOP/COEP em `next.config`/middleware (verificar que continuam ausentes).
- Mais lento → comunicar limites de tamanho/tempo na UI e exibir progresso.

## Grupo A — Cortar imagem (Image) — `crop-image`
Lib: `react-easy-crop` + `lib/image.ts`. Dropzone (jpg/png/webp) → área de recorte interativa
(`<Cropper>`), com aspecto livre e presets (1:1, 16:9, 4:3) opcionais → ao confirmar, desenhar
a região recortada num canvas (`drawToCanvas` com origem/dimensão do crop) → `canvasToBlob`
mantendo o formato (fundo branco se JPG) → preview + download. Reusa o padrão de
`image-converter`. **Sem dep de WASM — bom aquecimento.**

## Grupo B — Comprimir PDF (PDF) — `compress-pdf`
Lib: `pdfjs` + `pdf-lib` + `lib/image.ts` (**zero dep nova**). Dropzone 1 PDF → rasterizar
cada página com `pdf.js` (`render({canvas,viewport})`) num `scale` derivado da qualidade
escolhida → `canvasToBlob('image/jpeg', q)` → remontar um novo PDF com `pdf-lib`
(`embedJpg`, 1 página por imagem) → medir antes/depois (`formatBytes`) + download. Slider de
qualidade (default ~0.6–0.7). **Ressalva clara na copy:** é **compressão básica**; o texto
deixa de ser selecionável (páginas viram imagem). Guarda extra: se o resultado ficar **maior**
que o original, avisar e oferecer o arquivo original.

## Grupo C — Removedor de fundo (Image) — `background-remover`
Lib: `@imgly/background-removal`. Dropzone 1 imagem → roda o modelo IA **on-device**; **no
primeiro uso baixa ~vários MB** do modelo → **exibir progresso e aviso** ("first run downloads
the AI model"). Processador via `next/dynamic` (`ssr:false`, já é o padrão). Saída = **PNG com
transparência**; preview sobre fundo quadriculado; download. Comunicar que roda localmente
(privacidade) e que imagens grandes são mais lentas.

## Decisões default já embutidas (ajustáveis)
- **compress-pdf:** abordagem **rasterizar páginas** (compressão básica; perde texto selecionável),
  qualidade JPEG default ~0.6–0.7; só entrega se reduzir o tamanho.
- **ffmpeg core:** single-thread carregado via `toBlobURL` de CDN **pinado**; opção de self-host
  em `/public` se preferir não depender de CDN.
- **background-removal:** modelo padrão da lib (CDN); opção de self-host depois.
- **crop-image:** `react-easy-crop`; saída mantém o formato de origem (fundo branco p/ JPG).
- **mp4-to-mp3:** MP3 192 kbps; **audio-converter:** MP3/WAV/OGG/M4A com bitrate padrão sensato.
- **Guarda de tamanho (ffmpeg + background remover):** bloquear arquivos acima de um teto na UI
  (default sugerido: vídeo ~150 MB / áudio ~50 MB / imagem ~25 MB), com mensagem clara, para
  evitar travar a aba por falta de memória. O teto exato sai do spike (passo 0).

## Riscos detalhados e mitigações

Legenda de severidade: 🔴 alto · 🟠 médio · 🟢 baixo.

### Vídeo/Áudio (ffmpeg.wasm) — o grupo mais arriscado

| # | Risco | Sev. | Mitigação |
|---|---|---|---|
| V1 | **AdSense vs. multithread.** O ffmpeg rápido usa `SharedArrayBuffer`, que exige `COOP: same-origin` + `COEP: require-corp`. Esses headers quebram recursos cross-origin — **incluindo AdSense e analytics** — derrubando a monetização (a razão de existir do produto). | 🔴 | Usar **só** o core **single-thread** (`@ffmpeg/core`, nunca `-mt`). **Nunca** adicionar COOP/COEP no `next.config`/middleware. Documentar como footgun: trocar para multithread quebra os anúncios **silenciosamente**. |
| V2 | **Performance single-thread.** Sem paralelismo, transcodificar é lento (segundos a minutos). O custo "zero servidor" migra para a CPU/RAM/bateria do usuário. | 🟠 | Rodar em Web Worker; barra de progresso obrigatória; comunicar na UI que é local e pode demorar; sugerir arquivos curtos. |
| V3 | **Memória / crash com arquivos grandes.** WASM é 32-bit (heap prático ~1–2 GB). Entrada + saída + buffers podem **estourar a memória e travar a aba** sem erro limpo. | 🔴 | **Guarda de tamanho máximo na UI** (ver Decisões default) — praticamente obrigatória, não opcional. |
| V4 | **Cobertura de codecs/formatos.** O que o core lê/grava depende do build; MP3 (libmp3lame) costuma vir, mas **M4A/AAC** pode ser limitado/experimental. | 🟠 | **Spike do passo 0**: validar cada formato prometido antes da copy de SEO. Só prometer na UI o que o core realmente faz. |
| V5 | **Bundling WASM no Next 16/Turbopack.** O `@ffmpeg/ffmpeg` carrega worker + `.wasm`; bundlers às vezes brigam com isso. | 🟠 | Carregar o core via `toBlobURL` de **CDN pinado** em vez de empacotar o `.wasm`. Verificar no build (passo 0). |
| V6 | **Safari/iOS.** Limites de memória mais apertados → mais chance de crash no celular. | 🟡 | Tetos de tamanho mais conservadores no mobile; incluir Safari/iOS no smoke test. |

### Removedor de fundo (@imgly)

| # | Risco | Sev. | Mitigação |
|---|---|---|---|
| B1 | **Download do modelo no 1º uso** (~vários MB a dezenas de MB). | 🟡 | Progresso + aviso ("first run downloads the AI model"); versão pinada; tratar erro de rede. |
| B2 | **Performance/qualidade.** Inferência pesada em imagens grandes; recorte imperfeito em cabelos/bordas. | 🟡 | Guarda de tamanho de imagem; posicionar como "ótimo para a maioria dos casos", sem prometer perfeição. |

| P1 | **Rasterização perde texto selecionável** (acessibilidade/SEO do conteúdo) e pode **aumentar** PDFs só-texto. | 🟠 | Rotular como "compressão básica"; medir antes/depois; **só entregar se reduzir** — senão avisar e devolver o original. Alinhar com o usuário se a perda de texto é aceitável. |

### Cortar imagem

| # | Risco | Sev. | Mitigação |
|---|---|---|---|
| C1 | Imagens enormes consomem memória. | 🟢 | Guarda de tamanho de imagem; canvas puro reusando `lib/image.ts`. |

### Transversais

| # | Risco | Sev. | Mitigação |
|---|---|---|---|
| T1 | **Dependência de CDN no 1º uso** (core ffmpeg / modelo IA). CDN fora do ar ou bloqueado (rede corporativa/país) → ferramenta falha. | 🟡 | Pinar versão; erro amigável no carregamento; opção de **self-host** em `/public` depois (custa banda própria, remove a dependência). |
| T2 | **Peso de bundle.** Libs grandes (ffmpeg/imgly) não podem entrar no bundle inicial nem afetar outras páginas/SEO. | 🟢 | Manter atrás de `dynamic(ssr:false)` (já é o padrão); páginas seguem SSG e leves. |
| T3 | **Custo migra para o usuário** (CPU/RAM/banda), não para servidor. | 🟢 | É o trade-off aceito do modelo zero-custo; comunicar "runs locally" como benefício de privacidade. |
| T4 | **Regressão de monetização.** Qualquer header de isolamento cross-origin adicionado depois quebra o AdSense. | 🔴 | Checagem anti-regressão no `npm run build` (ver Verificação): garantir que `next.config`/middleware **não** emitem COOP/COEP. |

### Incógnitas que só se resolvem construindo (por isso o spike no passo 0)
- Suporte real de **M4A/AAC** no core escolhido.
- Comportamento do **Turbopack** com o worker/`.wasm` do ffmpeg.
- Números reais de **performance/memória** → definem o teto de tamanho.
