## Start here — handoff para uma nova sessão

**Como começar:** abra a nova sessão **nesta pasta do projeto** e peça "implemente o Passo 0 +
Grupo A deste plano". A memória do projeto (`MEMORY.md` + memórias linkadas) carrega
automaticamente e já traz: stack, motor config-driven, design Apple e que o **produto é em
inglês** (conteúdo/slugs em inglês; o usuário conversa em pt-BR).

**Leia estes arquivos primeiro (são os padrões a copiar):**
- `lib/tools/types.ts` — tipos `Tool` / `CategoryMeta` (o shape de cada ferramenta).
- `lib/tools/registry.ts` — copie a estrutura de um `Tool` existente (ex.: `merge-pdf`):
  metaTitle/description, h1, intro, **5 `sections` (≥800 palavras no total, em inglês)**,
  **4 `howTo`**, **5–6 `faq`**, `accept`/`multiple`.
- `components/tools/merge-pdf/merge-pdf-tool.tsx` e `image-compressor/...` — processadores
  client com arquivo (usam `FileDropzone`, `lib/download.ts`, `lib/format.ts`, toasts `sonner`).
- `components/tools/url-shortener/url-shortener-tool.tsx` e `text-rewriter/...` — processadores
  sem arquivo (input/textarea).
- `components/tools/tool-processor.tsx` — onde registrar cada tool: `dynamic(ssr:false)` +
  um `case "<slug-em-inglês>"` no switch.
- `components/tools/file-dropzone.tsx`, `lib/download.ts`, `lib/format.ts` — helpers a reusar.
- `DESIGN-apple.md` + `app/globals.css` — tokens (Action Blue, pílulas, Inter); a UI deve seguir.

**Convenções / pegadinhas:**
- shadcn aqui é **Base UI** (base-nova): `Button` usa `render` (não `asChild`); `Accordion` usa
  prop `multiple`. Para link estilizado de botão use `buttonVariants` num `<Link>`.
- Cada categoria com ferramenta aparece sozinha na home (showcase), no footer e em
  `/category/[slug]` — não precisa fiar nada manual além do registry + switch.
- Tudo client-side nesta onda; **Grupo A não precisa de nenhuma env** (Supabase/AI já estão no
  `.env.local`, mas só URL Shortener e Rewriter usam).

**Loop de verificação:** após cada ferramenta/grupo, rode `npm run build` (gera a página como
SSG) e `npm run lint`; teste no navegador que a ferramenta produz/baixa o resultado. SEO
(title/canonical/JSON-LD/sitemap) é automático pelo motor.

## Padrão por ferramenta (relembrando o motor)
Cada ferramenta = **1 objeto em `lib/tools/registry.ts`** (slug, categoria, SEO,
conteúdo pt-BR ≥800 palavras, `faq`, `howTo`, `accept`/`multiple`) **+ 1 processador**
em `components/tools/<slug>/` ligado no `switch` de `components/tools/tool-processor.tsx`
(lazy `next/dynamic`, `ssr:false`). Reaproveitar `FileDropzone`, `lib/download.ts`,
`lib/format.ts`. Página/metadata/sitemap/JSON-LD saem automáticos.

## Passo 0 — Dependências e base compartilhada
1. **Instalar:** `qrcode` + `@types/qrcode`, `pdfjs-dist`, `jszip`. (`pdf-lib` já existe.)
2. **`lib/image.ts`** (helper de canvas reaproveitável): `loadImage(file)` →
   `HTMLImageElement`; `drawToCanvas(img, {width,height})`; `canvasToBlob(canvas, type, quality)`.
   Usado por conversor-de-imagem, redimensionar-imagem (e futuros crop/remover-fundo).
3. **pdf.js worker:** configurar `GlobalWorkerOptions.workerSrc` via
   `new URL("pdfjs-dist/build/pdf.worker.min.mjs", import.meta.url)` num módulo client
   (`lib/pdfjs.ts`). Confirmar bundling no Turbopack.
4. **`components/ui/copy-button.tsx`** (opcional, pequeno): extrair o padrão de "copiar +
   feedback" já usado no encurtador/reescritor, para os tools de texto/web.

## Grupo A — Texto/Web puro JS (mais rápidos, sem arquivo)

| # | Ferramenta | slug | Cat. | Lógica |
|---|---|---|---|---|
| 1 | Contador de palavras | `contador-de-palavras` | Texto e IA | textarea → conta palavras/caracteres/frases/parágrafos + tempo de leitura (~200 ppm), ao vivo |
| 2 | Gerador de senha | `gerador-de-senha` | Web | opções (tamanho, maiúsc./minúsc./números/símbolos) → `crypto.getRandomValues` → saída + copiar |
| 3 | Formatador de JSON | `formatar-json` | Web | `JSON.parse` → `JSON.stringify(obj,null,2)` (formatar) / minificar; erro com mensagem; copiar |
| 4 | Codificador Base64 | `base64` | Web | toggle codificar/decodificar; **UTF-8 seguro** via `TextEncoder`/`TextDecoder` (não usar `btoa` cru) |
| 5 | Gerador de QR Code | `gerador-qr-code` | Web | input texto/URL → `QRCode.toCanvas` → preview + baixar PNG |

Esses não usam `FileDropzone`; UI = input/textarea + resultado + copiar/baixar.

## Grupo B — Imagem via canvas (`lib/image.ts`)

| # | Ferramenta | slug | Lógica |
|---|---|---|---|
| 6 | Conversor de imagem | `conversor-de-imagem` | dropzone (jpg/png/webp) → seletor de formato de saída (PNG/JPG/WebP) + slider de qualidade p/ lossy → `canvasToBlob` → baixar |
| 7 | Redimensionar imagem | `redimensionar-imagem` | dropzone → inputs largura/altura + travar proporção → canvas → baixar (mantém formato) |

## Grupo C — PDF via `pdf-lib`

| # | Ferramenta | slug | Lógica |
|---|---|---|---|
| 8 | Imagem para PDF | `imagem-para-pdf` | dropzone múltiplas imagens → `embedJpg/embedPng` (WebP/outros: converter p/ PNG via `lib/image.ts` antes) → 1 página por imagem → baixar PDF |
| 9 | Dividir PDF | `dividir-pdf` | dropzone 1 PDF → 2 modos: **extrair intervalo** (X–Y → 1 PDF) e **separar todas as páginas** (N PDFs → `.zip` via `jszip`) |

## Grupo D — PDF via `pdf.js`

| # | Ferramenta | slug | Lógica |
|---|---|---|---|
| 10 | PDF para JPG | `pdf-para-jpg` | dropzone 1 PDF → `pdf.js` renderiza cada página em canvas → `toBlob('image/jpeg')`; 1 página = baixar JPG, várias = `.zip` (`jszip`) |

## Sequência de execução
1. Passo 0 (deps + helpers).
2. Grupo A (5 tools) — registry + processador + wire, um a um.
3. Grupo B (2 tools).
4. Grupo C (2 tools).
5. Grupo D (1 tool).
6. Build + lint após cada grupo; smoke test no navegador.

> **Maior custo real = conteúdo SEO** (≥800 palavras + FAQ + HowTo pt-BR por ferramenta).
> Escrever junto com cada processador, não deixar acumular.

## Verificação (por ferramenta e por grupo)
- `npm run build` gera a página como SSG; `npm run lint` limpo; `tsc` ok.
- Browser: a ferramenta produz o resultado correto e baixa/copies o output.
- SEO automático conferido no HTML (title/canonical/JSON-LD/sitemap) — herdado do motor.
- Cada categoria afetada aparece corretamente na home/footer/`/categoria/<slug>`.

## Decisões default já embutidas (ajustáveis)
- **QR Code:** saída PNG (SVG fica como extra futuro).
- **Dividir PDF:** suporta intervalo único **e** separar-tudo em `.zip`.
- **PDF para JPG:** saída JPG (qualidade ~0.92); múltiplas páginas viram `.zip`.
- **Conversor/Redimensionar:** mantêm metadados mínimos; sem upload (tudo no navegador).
